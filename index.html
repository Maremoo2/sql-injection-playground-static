SQL Injection Playground — simulert (sikkert)

Dette er en klient-side simulering. Ingen ekte database eller server brukes. Trygt å publisere offentlig.

Prøv dette: Skriv et riktig passord (f.eks. wonderland) for å logge inn, eller eksperimenter med et alltid-sant uttrykk (f.eks. 2+2 = 4 eller ' OR '1'='1) for å se at den sårbare simuleringen omgår innlogging. Et galt passord vil bli nektet av den sikre sjekken.

Tips: Åpne DevTools → Console for å se den konstruerte SQL-strengen (nyttig for tekniske brukere).

Hvordan denne demoen virker

      
    
        
  1. Knappen «Sårbar» viser hvordan en applikasjon som setter sammen brukerinput direkte kan bygge sårbar SQL. Demoen simulerer hva som skjer hvis databasen evaluerer WHERE-betingelsen — den oppdager alltid-sanne uttrykk som matematiske likninger (2+2=4) og typiske injeksjonsmønstre (' OR '1'='1).
    2. 
        
  2. Knappen «Sikker» demonstrerer en korrekt sjekk: den sammenligner input mot en fast bruker-liste i nettleseren og tolker ikke input som SQL.
    3. 
      

      
Try inputs such as:

      
    
        
  • alice / wonderland — gyldige brukerdetaljer (suksess i både sårbar og sikker sjekk)
    • 
        
  • alice / ' OR '1'='1 — sårbar simulering kan omgå, sikker vil feile
    • 
        
  • bob / builder — gyldige brukerdetaljer
    • 
        
  • whatever / 2+2 = 4 — demonstrerer matematisk bypass i simuleringen

        Avansert (regexer og deteksjon)
        
Demoen bruker enkle mønstre for å oppdage alltid-sanne uttrykk i den konstruerte WHERE-klause. Kortversjon:

        
    
          
  • Aritmetikk: mønstre som a + b = c (støtte for +, -, *, /). Demoen evaluerer venstre side numerisk og sammenligner med høyre side; hvis lik, oppdages bypass (f.eks. 1+1=2).
    • 
          
  • Likhet: enkle likheter som 1 = 1 blir også oppdaget.
    • 
          
  • OR-mønster: klassiske injeksjonsmønstre som ' OR '1'='1 blir funnet med en enkel regex.
    • 
        

        
Merk: dette er for demonstrasjon. Virkelige SQL-parsere er mer komplekse, og denne demoen bruker forenklede regler for å gjøre poenget tydelig.
