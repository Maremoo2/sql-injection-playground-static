How this demo works

      
    
        
  1. The "Vulnerable" button shows how an application that concatenates inputs builds SQL. The demo then simulates what would happen if the DB evaluated the WHERE clause — it detects always-true expressions like math equalities (2+2=4) and typical injection patterns (' OR '1'='1).
    2. 
        
  2. The "Safe" button demonstrates a correct check: it compares inputs to a fixed in-browser user list and does not parse or evaluate the input as SQL.
